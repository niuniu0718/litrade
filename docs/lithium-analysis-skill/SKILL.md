# 锂资源分析框架

## 总纲

本Skill融合机构研报方法论、经典大宗商品分析框架、量化概率思维、以及实战采购经验，为锂资源采购与套保决策提供系统化分析支持。

## 分析流程

收到数据后，按以下步骤执行：

### Step 1：数据录入与校验

接收并整理以下数据（有哪些用哪些，缺失的标注）：

| 类别 | 指标 | 频率 |
|------|------|------|
| 价格 | SMM电池级碳酸锂均价、氢氧化锂均价、期货收盘价、锂精矿SC6 CIF | 日/周 |
| 供给 | 国内碳酸锂周度产量、进口量（智利/阿根廷）、开工率 | 周/月 |
| 需求 | 新能源车产销、正极材料排产（磷酸铁锂/三元）、储能装机 | 月 |
| 库存 | 社会库存总量、库存天数、广期所仓单、冶炼厂库存 | 周 |
| 成本 | 锂精矿价格、加工费（TC）、汇率、能源成本 | 周 |
| 持仓 | 期货持仓量、多空比、升贴水 | 日 |
| 事件 | 矿端政策、地缘冲突、天气、环保督查等 | 随时 |

### Step 2：逐模块分析

按顺序加载并执行各模块分析：

1. **供需平衡** → 读 `references/supply-demand-framework.md`
2. **库存周期** → 读 `references/inventory-cycle.md`
3. **成本支撑** → 读 `references/cost-support.md`
4. **事件驱动** → 读 `references/event-driven.md`
5. **需求景气度** → 读 `references/demand-outlook.md`

每个模块输出一个方向判断：**偏多 / 中性 / 偏空**，附简短理由。

### Step 3：综合评分与价格预测

将各模块判断汇总为综合评分，然后读 `references/price-forecasting.md` 执行多情景价格预测。

输出格式：

| 情景 | 概率 | 价格区间 | 触发条件 |
|------|------|---------|---------|
| 上行 | X% | XX-XX万 | ... |
| 基准 | Y% | XX-XX万 | ... |
| 下行 | Z% | XX-XX万 | ... |

**预计均价**：XX.X万元/吨

### Step 4：套保策略建议

读 `references/hedging-strategy.md`，结合用户持仓和采购计划，给出具体操作建议。

### Step 5：综合成本测算

读 `references/procurement-cost.md`，计算综合采购成本与SMM均价折扣。

### Step 6：输出报告

根据场景选择输出格式：

| 类型 | 触发条件 | 输出内容 |
|------|---------|---------|
| **日报** | 每日例行 | 价格变动+关键事件速览（200字以内） |
| **周报** | 每周或用户发周报时 | 完整Step 1-5分析 |
| **月报** | 每月或用户要求 | 长周期展望+采购策略 |
| **事件快评** | 突发事件发生 | 事件影响评估（Step 4+价格修正） |

## 分析原则

1. **数据驱动**：结论必须有数据支撑，避免纯主观判断
2. **多情景思维**：永远给出至少3个情景和概率权重，不做单点预测
3. **边际变化优先**：关注环比变化而非绝对水平，趋势比位置更重要
4. **区分实质与情绪**：事件影响分三级（实质性影响/短期情绪/无影响）
5. **概率思维**：用概率而非确定性表述，高置信度和低置信度要明确区分
6. **实战导向**：所有分析最终服务于采购决策和套保操作，必须给出可执行建议
